Q1.Differences between arrow function and regular function

Ans:1.
Arrow functionThe value of this in case of arrow function is considerably different from that of regular function. Arrow function doesn't define it's own context of execution. The value of this inside an arrow function is the same as that of it's outer function. In other words, arrow function resolve the value of this lexically and this behavior is independent of how and where the function is executed.

Let is a major improvement from the var keyword. Variables declared with the let keyword are block scoped. A block is any code written in between the curly brackets — { }. Block scoped variables are only accessible within that block.

Const is used for variables with a constant value. The value of the const variable will never change throughout the lifetime of the program. Just like let, const is also block scoped, i.e. it is restricted to the block in which it is declared. When declaring variables with const, it is necessary to initialize it with a value. In other words, arrow function resolve the value of this lexically and this behavior is independent of how and where the function is executed.

Regular function Inside a regular function, this value (i.e. the context of execution) is dynamic, depends on how the function is invoked. During simple invocation, the value of this is equal to the global object. During method invocation, the value of this is equal to the object owning it. During constructor invocation, the value of this refers to the newly created instance.
